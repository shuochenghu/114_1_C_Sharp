命名規範說明
================================================

1. 所有控件名稱均使用 camelCase（駝峰式命名）

2. 命名格式：描述性前綴 + 控件類型後綴
   例如：oilChangeCheckBox = oilChange（描述） + CheckBox（類型）

3. 命名原則：
   - 清晰表達控件的用途或所代表的數據
   - 使用英文命名，保持代碼的國際化
   - 避免使用數字編號（如 button1, label1 等）
   - 保持命名的一致性和可讀性

4. 常用控件類型後綴：
   - Button      → 按鈕
   - CheckBox    → 複選框
   - TextBox     → 文字框
   - Label       → 標籤
   - GroupBox    → 群組框

5. 特殊命名：
   - xxxValueLabel  → 表示顯示數值的標籤
   - xxxLabel       → 表示顯示文字標題的標籤

================================================
優點
================================================

1. 提高代碼可讀性：
   - 看到控件名稱就能立即知道其用途
   - 減少查找控件定義的時間

2. 便於維護：
   - 修改功能時容易找到相關控件
   - 降低出錯機會

3. 團隊協作：
   - 新成員能快速理解代碼結構
   - 統一的命名風格提高代碼質量

4. 智能感知支持：
   - IDE 的自動完成功能更有效
   - 輸入部分名稱即可快速找到控件
