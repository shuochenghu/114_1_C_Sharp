================================================
事件處理方法重命名前後對比
================================================

本文檔展示事件處理方法重命名前後的詳細對比

================================================
對比 1：calculateButton (計算總額按鈕)
================================================

【重命名前】
```csharp
// Form1.cs
/// <summary>
/// 計算總額按鈕事件
/// </summary>
private void button1_Click(object sender, EventArgs e)
{
    // 計算零件和工時（確保 labor 已計算）
    OtherCharges();
    // 顯示服務和工時總額
    serviceLaborValueLabel.Text = ...;
    // ...
}

// Form1.Designer.cs
this.calculateButton.Click += new System.EventHandler(this.button1_Click);
```

問題：
? button1_Click 名稱無意義
? 需要查看註解才知道功能
? 與控件名稱不一致

【重命名後】
```csharp
// Form1.cs
/// <summary>
/// 計算總額按鈕事件
/// </summary>
private void calculateButton_Click(object sender, EventArgs e)
{
    // 計算零件和工時（確保 labor 已計算）
    OtherCharges();
    // 顯示服務和工時總額
    serviceLaborValueLabel.Text = ...;
    // ...
}

// Form1.Designer.cs
this.calculateButton.Click += new System.EventHandler(this.calculateButton_Click);
```

優點：
? calculateButton_Click 清楚表達功能
? 與控件名稱 calculateButton 一致
? 方法名稱自我說明
? 容易搜尋和定位

對比 2：clearButton (清除按鈕)
================================================

【重命名前】
```csharp
// Form1.cs
/// <summary>
/// 清除按鈕事件，重設所有選項和輸入
/// </summary>
private void button2_Click(object sender, EventArgs e)
{
    // 清除所有選擇框和文字框
    ClearOilLube();
    ClearFlushes();
    ClearMisc();
    ClearOther();
    ClearFees();
}

// Form1.Designer.cs
this.clearButton.Click += new System.EventHandler(this.button2_Click);
```

問題：
? button2_Click 無法表達清除功能
? 編號 2 沒有任何意義
? 不符合命名慣例

【重命名後】
```csharp
// Form1.cs
/// <summary>
/// 清除按鈕事件，重設所有選項和輸入
/// </summary>
private void clearButton_Click(object sender, EventArgs e)
{
    // 清除所有選擇框和文字框
    ClearOilLube();
    ClearFlushes();
    ClearMisc();
    ClearOther();
    ClearFees();
}

// Form1.Designer.cs
this.clearButton.Click += new System.EventHandler(this.clearButton_Click);
```

優點：
? clearButton_Click 立即知道是清除功能
? 與按鈕文字「清除」對應
? 與控件名稱完全一致
? 符合 Windows Forms 慣例

對比 3：exitButton (離開按鈕)
================================================

【重命名前】
```csharp
// Form1.cs
/// <summary>
/// 離開按鈕事件，關閉表單
/// </summary>
private void button3_Click(object sender, EventArgs e)
{
    // 詢問是否要儲存維修明細
    DialogResult result = MessageBox.Show(
        "是否要儲存維修明細到檔案？", 
        "儲存確認", 
        MessageBoxButtons.YesNoCancel, 
        MessageBoxIcon.Question);

    if (result == DialogResult.Yes)
    {
        SaveServiceDetailsToFile();
    }
    else if (result == DialogResult.Cancel)
    {
        return; // 取消離開
    }

    this.Close();
}

// Form1.Designer.cs
this.exitButton.Click += new System.EventHandler(this.button3_Click);
```

問題：
? button3_Click 無法表達離開功能
? 看不出這是關閉應用程式的方法
? 維護時難以快速定位

【重命名後】
```csharp
// Form1.cs
/// <summary>
/// 離開按鈕事件，關閉表單
/// </summary>
private void exitButton_Click(object sender, EventArgs e)
{
    // 詢問是否要儲存維修明細
    DialogResult result = MessageBox.Show(
        "是否要儲存維修明細到檔案？", 
        "儲存確認", 
        MessageBoxButtons.YesNoCancel, 
        MessageBoxIcon.Question);

    if (result == DialogResult.Yes)
    {
        SaveServiceDetailsToFile();
    }
    else if (result == DialogResult.Cancel)
    {
        return; // 取消離開
    }

    this.Close();
}

// Form1.Designer.cs
this.exitButton.Click += new System.EventHandler(this.exitButton_Click);
```

優點：
? exitButton_Click 明確表示是離開功能
? 與按鈕功能完全對應
? 容易辨識關鍵的應用程式關閉邏輯
? 新開發者一看就懂

重命名前後總覽
================================================

【重命名前的程式碼結構】
```csharp
public partial class Form1 : Form
{
    // ... 欄位和建構函式 ...
    
    private void button1_Click(...)  // 這是什麼？
    {
        // 計算邏輯
    }
    
    private void button2_Click(...)  // 這又是什麼？
    {
        // 清除邏輯
    }
    
    private void button3_Click(...)  // 還是不知道做什麼
    {
        // 離開邏輯
    }
    
    // ... 其他方法 ...
}
```

問題：
? 需要逐一閱讀程式碼才知道每個方法的功能
? 方法名稱沒有提供任何有用資訊
? 增加認知負擔

【重命名後的程式碼結構】
```csharp
public partial class Form1 : Form
{
    // ... 欄位和建構函式 ...
    
    private void calculateButton_Click(...)  // 計算按鈕
    {
        // 計算邏輯
    }
    
    private void clearButton_Click(...)  // 清除按鈕
    {
        // 清除邏輯
    }
    
    private void exitButton_Click(...)  // 離開按鈕
    {
        // 離開邏輯
    }
    
    // ... 其他方法 ...
}
```

優點：
? 一眼就能理解每個方法的用途
? 方法名稱清楚表達功能
? 減少認知負擔
? 提高開發效率

- 需要逐一檢查確認

【重命名後】
搜尋「計算按鈕的事件處理」：
- 直接搜尋 "calculateButton_Click"
- 立即找到正確的方法
- 不需要額外確認

可讀性對比範例
================================================

【場景：新開發者閱讀程式碼】

重命名前：
```csharp
// 看到這三個方法，需要逐一閱讀才知道功能
private void button1_Click(...)  // ？
private void button2_Click(...)  // ？
private void button3_Click(...)  // ？
```

思考過程：
1. button1 是做什麼的？需要看程式碼
2. button2 又是什麼功能？繼續看程式碼
3. button3... 還要繼續看

時間消耗：需要閱讀 3 個方法的完整程式碼

重命名後：
```csharp
// 一看就知道每個方法的功能
private void calculateButton_Click(...)  // 計算
private void clearButton_Click(...)      // 清除
private void exitButton_Click(...)       // 離開
```

思考過程：
1. calculateButton_Click - 計算按鈕點擊
2. clearButton_Click - 清除按鈕點擊
3. exitButton_Click - 離開按鈕點擊

時間消耗：只需要看方法名稱

效率提升：約 80-90%
